MySQL System Requirements

Database: The minimal version is 5.30 for 1.9.3 and 5.1.x for 1.9.6 and higher. You should have a possibility to create functions on your MySQL hoster's server. Check it first.

CREATE FUNCTION in YAF might require the SUPER privilege to install functions, as described in Section 17.7, “Binary Logging of Stored Programs”.  Cheсk it befor installing.

The db object names should be case insensitive.

OS: Windows XP,Windows Vista,Windows Server 2003,Windows Server 2008.

.NET Framework: 4.0 for 1.9.6 Beta1 and higher and 2.0 for 1.9.3

My SQL INSTALLATION

STEP 1.Download and unpack archive.

  1. Download the archive from here . The files named like yaf_mysql_193. Releases like yaf_mysql_193-r1... - are simply bugfixes to main 1.9.3 release  and contain all the files from above version and a separate upgrade file and/or folder.
    For new YAF versions 1.9.6 Beta 1 and higher use source code from here. Installation details are mostly for the old version. For the version simply replace corresponding files according to folder structure. Delete /mssql folder in YAF.Classes.Data too. Configuration details are mostly common. You can use MySQL 5.1 and higher.
  2. Unzip the entire archive to a folder you want to run YAF in on your web server.

STEP 2. INSTALL/CHECK MySQL DB SETTINGS.

It supports MySQL 5.0.30 incl. 5.1.x. and 5.1.x. and higher beginning with 1.9.6 Beta. Other versions support is not garanteed.
Warning! Keep in mind that in versions 5.1.x  you should grant you user access to mysql db. If you receive error like: SELECT command denied to user 'yafuser'@'localhost' for table 'proc' use 5.0  MySQL version or give your user access to mysql database. In latest data layer options for 5.1.x in connection string in db.config you can use option
Use Procedure Bodies=false

 

Required connection string settings: Allow User Variables=True;(for all versions but it treated in data layer code.) Use Procedure Bodies=false;(for 5.1) Treat Tiny As Boolean=true;(provider default settings)

  Create new database yafnet(by default). All rights for your database should be granted to yafuser (by default) to simplify the install process. UTF-8 is surely supported and is default.
  The created user should have access to information_schema select and references priviliges and you should grant him all yafnet db priv. too. After installing you can drop a part of the privileges.
  Change db.config and app.config to your preferred settings.

STEP 4a. COMPILE YAF from source files (Optional for YAF 1.9.3)

If you have chosen this option than omit Step 4b.

Use /Source folder files. Replace all other files in included folders. Distribution folder contains ready code to simply replace it in a ready site. Source folder contains source code.

For YAF beginning with version 1.9.6 Beta 1 you should compile YAF from source code.

STEP 4b. INSTALL YAF MYSQL data layer(YAF 1.9.3 only).

If you have chosen this option than omit Step 4b.
  1. To use the data layer on existing site, simply add or replace files in /Distribution folder in respective folders. This contains ready files.
  2. Ajust your db.config file settings.
